## 🏗️ 架构设计

### 后端架构

```
Express.js Server (3001)
    │
    ├── Route Layer (20+ endpoints)
    │   └── Controllers (Request Processing)
    │
    ├── Service Layer (Business Logic)
    │   ├── UserService
    │   ├── PetService
    │   ├── MatchService
    │   ├── MessageService
    │   ├── NotificationService
    │   └── MarketService
    │
    ├── Middleware
    │   ├── Auth (JWT)
    │   ├── Error Handler
    │   └── CORS
    │
    ├── Database Layer
    │   └── Supabase (PostgreSQL)
    │
    └── Utils
        ├── JWT & Password
        └── API Client
```

### 数据库设计

```
核心表:
- users (用户账户)
- pets (宠物档案)
- matches (匹配记录)
- messages (消息系统)
- realms (社交空间)

支持表:
- user_stats (用户统计)
- market_items (市场商品)
- notifications (通知)
- reports (安全举报)

关系:
- users ←→ pets (一对多)
- pets ←→ matches (多对多)
- matches ←→ messages (一对多)
- users ←→ notifications (一对多)
```
