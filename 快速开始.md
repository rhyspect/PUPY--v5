# ⚡ 5分钟快速开始指南

## 🎯 目标

在5分钟内让前后端完整运行

## ✅ 前置条件（2分钟）

### 检查安装

```powershell
# 检查Node.js
node --version  # 应该是 v18 或更高

# 检查npm
npm --version

# 如果未安装，下载: https://nodejs.org
```

---

## 🔧 后端启动（2分钟）

### 步骤1: 进入后端目录

```powershell
cd "C:\Users\selin\Desktop\PUPY-爪住\后端V2"
```

### 步骤2: 安装依赖

```powershell
npm install
```

### 步骤3: 配置Supabase

1. 访问 [supabase.com](https://supabase.com)
2. 创建新项目
3. 获取项目URL和API密钥
4. 编辑 `.env` 文件：

```powershell
# Windows编辑.env
notepad .env
```

填入以下内容：
```env
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-public-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
JWT_SECRET=pupy_secret_key_min_32_chars_required_12345
PORT=3001
NODE_ENV=development
CORS_ORIGIN=http://localhost:3000,http://localhost:5173
```

### 步骤4: 启动后端

```powershell
npm run dev
```

**预期输出**：
```

║   🐾 PUPY Backend Server Started 🐾    ║
╠════════════════════════════════════════╣
║   Environment: development             ║
║   Port: 3001                           ║
║   API: http://localhost:3001           ║
║   Database: Supabase ✓                 ║
╚════════════════════════════════════════╝
```

✅ **后端启动成功！** 在另一个终端继续

## 🐛 常见问题

### ❌ 错误: "无法连接到Supabase"

**解决**:
1. 检查.env中的SUPABASE_URL是否正确
2. 确保Supabase项目已创建
3. 检查网络连接

### ❌ 错误: "CORS错误"

**解决**:
```env
# 在后端.env中添加
CORS_ORIGIN=http://localhost:5173
```

### ❌ 前端显示"网络错误"

**解决**:
1. 检查后端是否在运行: http://localhost:3001/api/health
2. 验证前端.env.local中的VITE_API_URL
3. 重启前端: Ctrl+C, 然后 npm run dev

### ❌ 端口已被占用

**解决**:
```powershell
# 更改后端端口
$env:PORT=3002
npm run dev

# 然后更新前端.env.local
# VITE_API_URL=http://localhost:3002/api
```
